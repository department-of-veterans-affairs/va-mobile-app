# User-Facing UI Changes: Feb 25 – Mar 11, 2026

This document summarizes all user-facing UI changes merged into the VA Mobile App repository between **February 25, 2026** and **March 11, 2026**. Changes are grouped by feature area. Feature flag dependencies are noted where applicable.

---

## Veteran Status Card (VSC)

### Adjusted Alert Message Spacing
**PR [#12835](https://github.com/department-of-veterans-affairs/va-mobile-app/pull/12835) · Merged Mar 11, 2026**

Spacing within the `veteran_status_alert` message on the Veteran Status Card was adjusted to better match the approved Figma design. The update improves readability by correcting the spacing between paragraphs and phone numbers in the alert.

- **Feature flag:** `veteranStatusCardUpdate` must be enabled
- **Affected screen:** Veteran Status Card — alert message section

---

### Hide DoD ID Field When Empty
**PR [#12819](https://github.com/department-of-veterans-affairs/va-mobile-app/pull/12819) · Merged Mar 6, 2026**

The DoD ID Number field on the Veteran Status Card is now hidden when the backend returns an empty value. Previously the field could appear with no data, creating a confusing UI state for the edge case where a Veteran's DoD ID is unavailable.

- **Feature flag:** `veteranStatusCardUpdate` must be enabled
- **Affected screen:** Veteran Status Card — identification fields

---

### Remove Period of Service Error Info
**PR [#12754](https://github.com/department-of-veterans-affairs/va-mobile-app/pull/12754) · Merged Feb 26, 2026**

Outdated period-of-service error information was removed from the Veteran Status Card. Content that was previously shown when period-of-service data had errors is no longer displayed.

- **Feature flag:** `veteranStatusCardUpdate` must be enabled
- **Affected screen:** Veteran Status Card — period of service section

---

## Benefits — Claims & Evidence

### Evidence Request Details: Enriched Default Content (FileRequestDetails)
**PR [#12783](https://github.com/department-of-veterans-affairs/va-mobile-app/pull/12783) · Merged Mar 6, 2026**

The `FileRequestDetails` screen now renders enriched default content when an evidence request does **not** include a `friendlyName` field (i.e., no content overrides are configured). When a `friendlyName` is present, the existing display is preserved.

**New UI sections for requests without a `friendlyName`:**
1. **Title:** "Request for evidence"
2. **Subtitle:** "Respond by {suspenseDate} for: {displayName}" (shown only when `suspenseDate` exists)
3. **Request date blurb:** "We requested this evidence from you on {date}…" (shown only when `requestedDate` exists; includes "respond by" reference only when `suspenseDate` is also present)
4. **What we need from you** — section header with `description` text
5. **How to submit this information** — instructions with links to claim letters and VA forms
6. **More on submitting files** — collapsible accordion with in-app, mail, and in-person submission options (hidden when `canUploadFile` is `false`)
7. **Need help?** — collapsible accordion with VA Benefits Hotline info and a click-to-call link (800-827-1000)
8. **Upload buttons** — "Select a file" and "Take or select photos" (shown when no files have been uploaded yet)

- **Feature flag:** `evidenceRequestsUpdatedUI` must be enabled
- **Affected screen:** Benefits → Claims → File Request Details

---

### Evidence Request List: Use `friendlyName` with `displayName` Fallback (FileRequest)
**PR [#12612](https://github.com/department-of-veterans-affairs/va-mobile-app/pull/12612) · Merged Feb 27, 2026**

The `FileRequest` list component (the evidence request list on a claim) was updated to use `friendlyName` as the primary label when content overrides are available. When no `friendlyName` exists, the component now shows **"Request for evidence"** as the primary text with `displayName` as a secondary label underneath, instead of displaying `displayName` alone.

| Condition | Display |
|-----------|---------|
| `friendlyName` present | `friendlyName` only |
| No `friendlyName` | "Request for evidence" (primary) + `displayName` (secondary) |

- **Feature flag:** `evidenceRequestsUpdatedUI` must be enabled
- **Affected screen:** Benefits → Claims → File Requests list

---

## Health — Secure Messaging

### Combobox Header Now Shows Selected Care Team Name
**PR [#12779](https://github.com/department-of-veterans-affairs/va-mobile-app/pull/12779) · Merged Feb 27, 2026**

On the **Start New Message** and **Edit Draft** screens, the "To" field's recipient picker (combobox) previously always displayed the header "All Care Teams." The header now shows the name of the **currently selected care team** instead, giving Veterans clearer context about which care system they are messaging.

- **Feature flag:** None (always on)
- **Affected screens:** Secure Messaging → Start New Message, Edit Draft

---

### Oracle Health Migration Alerts & Button Changes (Secure Messaging)
**PR [#12701](https://github.com/department-of-veterans-affairs/va-mobile-app/pull/12701) · Merged Mar 2, 2026**

A series of UI changes were made to support Oracle Health (OH) facility migration scenarios in Secure Messaging:

- **Inbox:** Displays a feature-level migration error message when all user facilities are in an error migration state.
- **View Message screen:** Migration-aware alerts (error/warning) are shown on individual message threads based on the sending facility's migration status. The **Reply** button is swapped for a **Start New Message** button when migration blocks replies.
- **Start New Message screen:** Red migration error alerts are displayed when the user's facilities are in an error migration state.
- **Edit Draft screen:** Red migration error alerts added for in-progress drafts when the facility is in a migration error state.
- **All folder views (Inbox, Drafts, Sent, Custom folders, empty state):** The **Start new message** (compose) button is hidden when all user facilities are in a migration error state or when no message recipients are available.

- **Feature flag:** OH migration feature flags (applied based on facility migration phase)
- **Affected screens:** Secure Messaging — Inbox, View Message, Start New Message, Edit Draft, all folder views

## Health — Medications / Prescriptions

### Oracle Health Transition Alerts on Prescription Details
**PR [#12735](https://github.com/department-of-veterans-affairs/va-mobile-app/pull/12735) · Merged Mar 9, 2026**

A new `PrescriptionDetailsBanner` component was added to the Prescriptions feature to surface Oracle Health (OH) cutover messaging:

- **Prescription Detail page:** For medications from transitioning OH facilities, the **Request Refill** button is hidden and the transition banner is displayed in its place. Medications from non-transitioning facilities continue to show the Request Refill button normally.
- **Refill Request screen:** Only medications from non-transitioning facilities appear as selectable options. A banner lists links to the Prescription Detail pages of medications from transitioning facilities that cannot currently be refilled.

- **Feature flag:** `mhvMedicationsOracleHealthCutover` must be enabled
- **Affected screens:** Prescriptions → Prescription Details, Refill Request

## Health — Medical Records

### Duplicate Records Alert for Oracle Health Users
**PR [#12756](https://github.com/department-of-veterans-affairs/va-mobile-app/pull/12756) · Merged Mar 5, 2026**

A new dismissible informational alert was added to the **Medical Records** screen to notify Oracle Health (OH) facility users that they may temporarily see duplicate records while backend de-duplication is in progress.

**Display logic:**
- Shown to users at an OH facility who are **not** currently in another active migration phase that already shows an alert on the Medical Records page.
- Also shown to users in migration phases P6 or P7 (T+2 or later) to account for potential lag in facility status updates.
- If the user is at an OH facility but also in another active migration phase, migration alerts take precedence and the duplicate records alert is not shown.
- The alert is **collapsible and dismissible**; once dismissed, it will not reappear on subsequent logins.

- **Feature flag:** None (logic based on facility type and migration phase)
- **Affected screen:** Health → Medical Records

## iOS Platform Fix

### Restore Missing Cancel Buttons (iOS 26)
**PR [#12750](https://github.com/department-of-veterans-affairs/va-mobile-app/pull/12750) · Merged Feb 26, 2026**

An upgrade to iOS 26 caused **Cancel** buttons to disappear from Action Sheets throughout the app. The iOS 26 redesign changed action sheets to render as floating cards and broke the `.cancel` style action rendering. Cancel buttons have been restored across the following screens:

- **Components:** File List, Photo Preview, Photo Upload, Full-Screen Subtask, Large Panel
- **Benefits:** Ask for Claim Decision, File 5103 Review Waiver, Select File, Upload File, Upload or Add Photos
- **Health:** Appointment Cancel/Reschedule, Prescription Details, Refill Screen, Compose Cancel Confirmation, Edit Draft, Interstitial Screen
- **Home / Profile:** Edit Address, Edit Email, Edit Phone Number, Preferred Name, Settings
- **Payments:** Copay Details, Resolve Copay, Resolve Debt, Edit Direct Deposit

- **Feature flag:** None (platform fix, iOS 26 only)
- **Affected platform:** iOS 26+
